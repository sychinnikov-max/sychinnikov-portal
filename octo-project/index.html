Операционная система
проекта Octo

Полный контур управления партнёрской компанией: воронка, продукт, экспертиза, команда, решения. Построен как самостоятельный передаваемый актив.

114 партнёров в базе 7 этапов воронки ~€10K средний чек 60–70% конверсия лид → продажа
О системе

Что это и зачем

Это не папка с документами. Это операционная среда компании, в которой каждый день формируются изменения, раскладываются по нужным местам, и AI-агент помогает работать с ними. Цель — превратить ежедневную операционку в передаваемый актив.

📈

Рост выручки сейчас

Узкое место — поток лидов от партнёров. Система фокусирует усилия на одном рычаге: позиционирование Octo + материалы для партнёрской интеграции.

Логика потока

Как работает система

Поток движется сверху вниз: от привлечения партнёра до передачи клиента в delivery. На каждом слое — свой owner, свои метрики, свой ритм.

1. ACQUISITION — Привлечение

Позиционирование Octo. Материалы. Скрипты. Узкое место — поток лидов.

↓

2. FUNNEL — 7 этапов воронки

Контакт → Квалификация → УТП → Подкаст → Маркетинг-разбор → Готов к запуску → Первый клиент ✅

↓

3. HANDOVER — Передача клиента

Лид уходит в команду Елены (Люба → кейс-менеджеры → юридический эксперт). Конверсия в продажу ~60–70%.

↓

4. Партнёр в активном статусе

Цикл повторяется со следующими клиентами. Партнёр зафиксирован в pipeline.

Поперечные слои: expertise (знание продукта), ops (люди, финансы, инструменты), command (решения). Они обслуживают весь поток — не привязаны к одному этапу.

Правила игры

Стандарт качества (b)

Каждый блок проектируется как самостоятельный передаваемый актив. Это не «папка для меня» — это «продукт, который можно отделить».

1
README в каждой папке. Что внутри, кому читать, когда обновляется, кто owner. Без README — папки не существует.
2
Изоляция данных. Public/sellable и private физически разделены. При отделении актива: rm -rf private/ — готово.
3
Объяснимо за 30 минут постороннему. Любой блок новый человек понимает, открыв 2–3 файла. Не понимает — баг структуры.
4
Naming convention — единая. Lowercase-через-дефис. Английские названия, русский контент. Никаких temp/ или v2/.
5
Артефакты ≠ состояние. state.md меняется ежедневно; шаблоны и плейбуки стабильны. Разные папки, разный ритм.
6
Audit-trail в decisions.md. Решение, влияющее на структуру/процесс — явная ADR-запись. «Договорились в чате» не считается.
7
Operator manual обязателен. Если блок претендует на sellable — есть инструкция. Без неё блок остаётся внутренней кухней.
