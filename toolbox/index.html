Toolbox

Полный набор инструментов AndreiOS: skills, субагенты, hooks и интересные команды.

Логика: Skills — атомарные процедуры по слэш-команде. Субагенты — специализированные «коллеги» с изолированным контекстом. Hooks — автоматические действия системы. Команды — точечные триггеры для запуска skills.

🛠 Skills — 8 шт.

Атомарные процедуры. Каждый skill решает одну задачу строго по своему сценарию.

📥 /intake

/i <текст> /intake <текст> «запиши в инбокс»

Микро-операция: добавить сырую запись в inbox/today.md с таймстампом. Не классифицирует, не анализирует, не комментирует. Просто записывает.

Когда: в течение дня, постоянно. Любая мысль / событие / цифра / решение / файл — сразу в инбокс. Чтобы не держать в голове.
📄 inbox/today.md ⚡ <500 токенов 🔁 десятки раз в день

💰 /finance новое

/finance /f «обед 18 евро картой» «пришла зарплата Fishok»

Учёт личных финансов по принципу план-факт. Записывает транзакцию (доход/расход/перевод), обновляет конверт, остаток счёта и общую картину. Показывает остатки, закрывает месяц.

Когда: при любом денежном событии. Цель — записать за 1-2 секунды и забыть. Skill сам распарсит сумму, категорию, счёт.
  • Запись свободным текстом: «такси 12 наличными», «купил Саре куртку 45», «пришла зарплата Fishok 3500»
  • Slash: /f — открыть состояние, /finance close — закрытие месяца, /finance budget — изменить план
  • Запросы: «сколько в конверте няня?», «сколько потратил на еду в мае?»
  • Дедупликация: для регулярных доходов (Fishok 14-го, Octo 1-го) проверяет последние 5 дней
📄 finance/state.md, finance/ledger.md, finance/budget.md ⚡ ~1-2K токенов 🔁 несколько раз в день

🌙 /day-close

/day-close «закроем день»

Закрытие дня — главный регулярный цикл. Разбирает накопленный inbox, классифицирует записи по проектам, обновляет state-файлы, пишет свод в daily/YYYY-MM-DD.md, архивирует inbox.

Когда: один раз вечером. Перед сном или после ужина — когда поток сырья закончился.
  • Запускает субагент business-operator для классификации
  • При денежных событиях подключает finance-director
  • Обновляет: state.md проектов, finance/state.md, decisions.md, family.md
  • Шаг 2a проверяет проектные inbox-ы (например clients/octo/inbox/)
  • Stop hook делает git commit автоматически
📄 daily/, state.md клиентов, finance/, decisions.md ⚡ ~10-20K токенов 🔁 1 раз / день

💭 /weekly-dialogue

/weekly-dialogue /weekly «закроем неделю»

Недельная рефлексия в формате диалога 60-90 минут. Не отчёт. Claude задаёт по одному вопросу за раз, копает глубже, спорит. Цель — родить новые смыслы, не просто записать факты.

Когда: пятница вечером или суббота утром.

8 блоков сессии: 💰 Финансы → 🐟 Fishok → 🐙 Octo → 📥 Pipeline → 👨‍👩‍👧 Семья → 🎯 Стратегия → 🧠 Личное → ✅ Синтез

  • Читает 7 последних daily/, all state.md, finance/state.md, money.md, decisions.md
  • Записывает weekly/YYYY-Wнн.md
  • Обновляет this-week.md на следующую неделю
  • В конце предлагает запустить context-architect для health-check
📄 weekly/, this-week.md, state.md ⚡ ~30-50K токенов 🔁 1 раз / неделю

🔄 /update-state

/update-state <путь> «обнови state по Fishok»

Точечное обновление state.md проекта вне регулярного цикла. Аварийный инструмент — для срочной фиксации изменения.

Когда: по запросу. Если используется чаще 1-2 раз в день — значит inbox + day-close не справляются, надо разбираться почему.
📄 state.md одного проекта ⚡ ~3-5K токенов 🔁 редко, по запросу

🐙 /octo-intake только Octo

/octo-intake «разбери Octo inbox»

Полноценный разбор сырья из clients/octo/inbox/ и разнос по нужным файлам проекта Octo. В отличие от /intake (микро-операция «положить в inbox»), это разбор с обновлением state-файлов проекта.

Когда: когда в octo/inbox/ накопилось сырьё. Обычно после звонков или серии записей про Octo.
  • Использует субагент octo-advisor для классификации в логике Octo
  • Обновляет: command/state.md, funnel/pipeline.md, ops/people/, command/decisions.md, command/issues.md
  • Никогда не лезет в другие клиентские папки
📄 clients/octo/* (всё кроме private/) ⚡ ~10-15K токенов 🔁 по запросу

🎭 /expert-panel новое

/expert-panel /panel «разбери с трёх позиций» «дай мнение трёх экспертов»

Прогнать предложенное решение / артефакт / структуру через панель из 3+ экспертных ролей. Каждый эксперт критикует мой вариант, предлагает свою версию и подсвечивает слабое место своего же подхода. Дальше — сравнительная таблица, моя честная позиция (что взять, что не брать), и опрос для поэлементного выбора.

Когда: для важных структурных решений до применения. Когда чувствую слепые зоны или хочу проверить себя.
📄 артефакт решения ⚡ ~5-10K токенов 🔁 для серьёзных решений

📐 /bpmn-generator новое

/bpmn /process «опиши процесс в BPMN» «построй BPMN для X»

Сгенерировать BPMN 2.0 диаграмму процесса в виде валидного XML, готового к импорту в bpmn.io / Camunda / draw.io.

Когда: когда нужно описать бизнес-процесс в стандартной нотации (для клиента, документации, обучения команды).
📄 BPMN 2.0 XML ⚡ ~3-5K токенов 🔁 по запросу

🧑‍💼 Субагенты — 4 шт.

Специализированные «коллеги» с собственным контекстом и моделью. Изолируют сложные задачи от основного диалога — экономят токены, дают чистый фокус.

🐙 business-operator

Sonnet

Операционный архитектор. Главный исполнитель по клиентским проектам и обработке inbox.

Когда вызывается:

  • В /day-close — классифицирует записи inbox по проектам, обновляет state-файлы
  • При работе с Fishok / Octo — помогает с операционкой
  • При создании регламентов, отчётов, оргструктур
  • При работе с воронками продаж и CRM
🔧 Read, Glob, Grep, Edit, Write, Bash 📋 Operational Foundation как фон

💰 finance-director

Sonnet

Семейный и проектный финдиректор. Cashflow, runway, долги, риски, инвестиции, equity-портфель.

Когда вызывается:

  • В /day-close если за день были денежные события
  • В /weekly-dialogue — раздел «Финансы»
  • При работе с money.md и финансовыми решениями
  • При обсуждении equity-сделок и инвестиций

Принципы: Деньги → Семья → Фокус. В каждом обновлении явно пишет runway X месяцев. Equity > Cash на 7-летнем горизонте.

🔧 Read, Glob, Grep, Edit, Write, Bash 📄 money.md, finance/, decisions.md

🏛 context-architect

Opus

Архитектор контекста AndreiOS. Не операционный исполнитель — «уборщик» системы. Чистит дубли, ищет противоречия, предлагает структурные изменения.

Когда вызывается: вручную, раз в неделю в /weekly-dialogue или когда чувствуется, что система раздувается хаотично. Также при срабатывании kill-switch — для ретроспективы.

Что делает: ищет дубли фактов, противоречия state.md ↔ decisions.md, устаревшие страницы (>14 дней), сиротские файлы, нарушения правила роста.

Не правит содержание файлов, только предлагает. Финальное решение всегда за Андреем.
🔧 Read, Glob, Grep, Edit, Write 🎯 цель: система остаётся управляемой через 6/12/24 месяца

🐙 octo-advisor только Octo новое

Opus

Внутренний советник проекта Octo Relocate. Архетип: опытный сооснователь/инвестор в mobility/relocation отрасли + AI engineer (context-engineering, Claude Code).

Когда вызывается: явно, для стратегических разборов, аудита системы Octo, помощи с улучшением проекта. Не для рутины — для глубокого мышления.

  • Работает только с папкой clients/octo/ — полная изоляция от остальной AndreiOS
  • Полный Read/Edit/Write по всему дереву Octo, кроме clients/octo/private/ (только чтение)
  • Используется в /octo-intake для классификации
  • Не видит Fishok, money.md, family.md и остальной общий контекст
🔧 Read, Glob, Grep, Edit, Write, Bash 🔒 изолирован от других клиентов

⚙️ Hooks и permissions

Автоматические действия системы и правила безопасности. Срабатывают без явных команд.

🪝 Stop hook — авто-коммит сессии

После каждой завершённой сессии Claude автоматически делает git commit изменений в репозитории. Так история всегда сохраняется без ручного «не забыть закоммитить».

cd "$CLAUDE_PROJECT_DIR" && git add -A && git commit -m "auto: $(date '+%Y-%m-%d %H:%M') — session end"

Зачем: каждая сессия = checkpoint. Если что-то сломается через 3 дня — можно вернуться. Плюс это входной сигнал в kill-switch (14 дней без коммита = система мёртвая).

🔒 Permissions — что разрешено и запрещено

DENY (запрещено всегда):
  • rm -rf в любом виде
  • sudo rm
  • чтение .env, *.key, *.pem
  • curl / wget с экспортом
ALLOW (разрешено автоматом):
  • git add/commit/status/log/diff/init
  • ls, mkdir, mv, cp, cat, echo, pwd
  • Read, Write, Edit любых файлов

Логика: разрешено всё что нужно для работы со state-файлами и git, запрещено всё что может удалить данные или утечь секреты.

⌨️ Полезные команды Claude Code — системные

Это не «мои» команды, а встроенные в Claude Code. Полезны для нашей системы.

/init

Создаёт CLAUDE.md в новой папке с описанием контекста. Использовать при создании нового клиентского проекта.

/review

Code review текущей ветки или PR. Для нашей системы — обзор изменений перед закрытием недели.

/loop <интервал> <команда>

Запустить команду повторно с интервалом. Например /loop 5m /day-close — но мы не используем, регулярность через day-close достаточна.

/schedule

Запланировать запуск задачи по расписанию (cron). Кандидат для авто-обновления дашборда раз в день.

/simplify

Ревью изменённого кода: дубли, упрощения, вытащить общее. Полезен при росте HTML-портала.

/update-config

Настроить hooks, permissions, env-переменные в settings.json. Через него добавляли Stop hook.

/fewer-permission-prompts

Сканирует историю сессий и предлагает добавить часто используемые команды в allow-list. Уменьшает число подтверждений.

/telegram:configure

Настроить Telegram-канал для общения с Claude. Полезно когда нужно кинуть мысль/файл с телефона в inbox.

🎯 Типичные паттерны использования

Обычный день

  1. В течение дня — /i <любая мысль> постоянно. Десятки раз. Не думать «куда положить» — всё в инбокс.
  2. Любая трата / зарплата → пишу как в чате: «обед 18 евро картой», «такси 12 наличными». Skill /finance сам разберёт.
  3. Вечером — /day-close. Один раз. Разбирает накопленное, обновляет state, пишет свод.
  4. Спать. Stop hook закоммитил.

Конец недели

  1. Сначала /day-close текущего дня (если ещё не сделан).
  2. Потом /weekly-dialogue — 60-90 минут разговора. Без спешки.
  3. В конце Claude предложит context-architect — соглашайся раз в 2-4 недели.

Конец месяца

  1. 1-го числа — /finance close. Закрытие месяца: сверка остатков, снимок в archive/, обнуление ledger.
  2. Если планы конвертов меняются (новые расходы, изменился доход) — обнови finance/budget.md через /finance budget.

Работа с Octo

  1. Сырьё/мысли по Octo → можно через общий /i (попадёт в inbox), или через clients/octo/inbox/.
  2. Накопилось много Octo-сырья → /octo-intake разберёт целенаправленно.
  3. Глубокая стратегическая сессия по Octo → вызывай агент octo-advisor явно. Он изолирован от остального — чистый фокус.

Когда что-то важное случилось «прямо сейчас»

  • Мелкое → /i <описание>, обработается вечером в day-close.
  • Срочное обновление state.md → /update-state clients/octo
  • Финансовый вопрос → просто работа с агентом finance-director в обычном диалоге.

Когда чувствуется хаос в системе

  1. Не создавай новые файлы / папки — это усилит хаос.
  2. Запусти context-architect — он покажет дубли, противоречия, сиротские файлы.
  3. По его предложениям — чистка. Только после твоего подтверждения.

Toolbox обновляется по мере роста системы. Каждый новый skill должен соответствовать правилу роста: повторяется ≥3 раз ИЛИ нужен для одной из трёх целей 90 дней.
