AndreiOS — карта системы

Концептуальная архитектура: фундамент, финансы, клиенты, AI-команда, циклы работы и правила.

Главный принцип: AndreiOS — это не «инструмент управления задачами». Это контекстная база, в которую раз в день стекаются события, а AI-команда помогает разложить их по проектам, обновить состояние и поставить приоритеты на завтра.

Чтение схемы: Андрей кидает сырьё в inbox в течение дня, любые денежные события — сразу в /finance. Вечером /day-close разбирает inbox и обновляет нужные файлы. Раз в неделю /weekly-dialogue переосмысляет состояние и обновляет план.

2. Фундамент — 5 файлов в корне

База. Меняется редко, но влияет на каждое решение.

  • CLAUDE.md Главная конституция-роутер. Читается перед каждым сообщением. Три цели 90 дней, правила контекста, что НЕ делать.
  • me.md Кто я, ценности, антицели, главная цель, двухтактная стратегия, фильтры решений
  • family.md Семейный снимок: миграция, дети, документы, договорённости с Викой
  • decisions.md Журнал стратегических решений с YAML-тегами scope (strategic/finance/family/fishok/octo/pipeline)
  • this-week.md Недельный план: P1/P2/P3 задачи, блоки по проектам, гипотеза недели

3. Финансы — отдельный слой

В мае 2026 финансы выделены в отдельную папку с системой план-факт. Раньше всё жило в money.md, теперь — три файла + skill для записи транзакций.

  • finance/state.md Живая картина месяца: план-факт по конвертам, остатки счетов, прогресс к подушке, сигналы. Обновляется при каждой записи транзакции.
  • finance/budget.md Структура 12 конвертов, шаблоны распределения зарплаты, правила фондов. Меняется редко (раз в месяц).
  • finance/ledger.md Append-only журнал транзакций текущего месяца. Архивируется в finance/archive/ при закрытии месяца.
  • finance/archive/ Закрытые месяцы: снимок плана-факта + полный ledger.
  • money.md (корень) Стратегический слой: cashflow по горизонтам (6/12 мес, 10 лет), долги, equity-портфель, фонды. Не оперативка.

Как записывать: просто «обед 18 евро картой» в чате — skill /finance разберёт сумму, категорию, счёт, дополнит state и ledger.

4. Клиенты — изолированные проекты

Каждый клиент — отдельная папка с собственной структурой. Никакого «общего» state — изоляция предотвращает утечки контекста.

🐟 Fishok текущий доход

Латвия. €3 500/мес. Производство натуральных рыбных лакомств, B2B. Стандартная структура: CLAUDE.md, state.md, private/shared, meetings, raw, work.

Тег в decisions.md: fishok

🐙 Octo Relocate главный equity-канал

Испания. €1К фикс + до €3К бонус. Релокационно-визовая компания. Расширенная структура из 7 верхних папок (см. полную карту): acquisition, funnel, handover, expertise, ops, command, private + work.

Тег: octo · детальная зона →

6. AI-команда — 4 субагента + 6 skills

Субагенты .claude/agents/

Специализированные «коллеги» с изолированным контекстом. Изолируют сложные задачи от основного диалога.

  • business-operator — операционка по клиентам
  • finance-director — финансовые решения
  • context-architect — чистка системы (раз в 2-4 нед)
  • octo-advisor — внутренний советник Octo (изолирован от других клиентов)

Skills .claude/skills/

Атомарные процедуры по слэш-команде.

  • /intake, /i — десятки раз в день
  • /finance, /f — учёт транзакций
  • /day-close — 1 раз вечером
  • /weekly-dialogue — пятница вечером
  • /update-state — по запросу
  • /octo-intake — разбор сырья Octo

7. Правила — что держит систему от деградации

Правило роста: новая папка или файл создаётся только когда (а) действующий перерос 200 строк, (б) появилось ≥3 однотипных артефакта, или (в) явное решение «нужно для одной из трёх целей 90 дней». Запрещено «на всякий случай».
Kill-switch: если 14 дней нет коммитов в репозитории — система признана мёртвой. 30 дней не строим следующую итерацию. Сначала ретроспектива «почему не работало». Текущее состояние таймера →
Изоляция контекста: при работе с проектом — только его файлы. При работе с мини-проектом внутри клиента (clients/octo/work/website/) — только локальные файлы, не весь клиент. Экономия токенов и фокус. octo-advisor физически не видит fishok/.
Не читать без явного запроса: archive/ (старые версии), inbox/today.md вне /day-close, private/ клиентов в публичных артефактах.
Append-only для decisions: ADR никогда не редактируются. Если решение изменилось — пишется новый ADR со ссылкой на старый. История мышления сохраняется.

8. Как улучшать систему

Три направления роста, в порядке приоритета:

  1. Наполнение state-файлов клиентов. Сейчас Fishok ещё частично пуст — наполнится через первые /day-close и /weekly-dialogue.
  2. Новые skills под повторяющиеся операции. Если задача делается ≥3 раз — превращать в skill. Кандидаты сейчас: /dashboard-update, /client-onboarding, /decision-add.
  3. Новые субагенты под равные задачи. Когда новый клиент потребует своего advisor — добавится. Кандидат: agent для анализа звонков из meetings/.
