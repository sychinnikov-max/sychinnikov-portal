Принципы работы с клиентами

Изоляция контекста: у каждого клиента своя папка clients/<name>/ с собственной структурой. Никакого «общего» state. Octo-advisor (специальный субагент) не видит Fishok — и наоборот. Это экономит токены и предотвращает утечки.

Стандартная структура:

  • CLAUDE.md — контекст проекта
  • state.md с YAML frontmatter — оперативная память
  • private.md или private/ — моя разведка, НЕ передаётся при экзите
  • shared.md — что ОК показать команде клиента
  • touches.md — лог касаний (для проспектов)
  • raw/ — транскрипты, PDF, исходники
  • work/ — мини-проекты с собственным жизненным циклом

Octo — продвинутый вариант (7 верхних папок, см. Octo OS →). Используется как шаблон для крупных клиентов.

Каждый клиент имеет state.md с YAML frontmatter. Pipeline.md автогенерируется из state-файлов в /day-close.
